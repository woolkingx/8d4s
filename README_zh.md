# 8D4S：讓 AI 擁有反思能力的思考框架

> *「不是讓 AI 變聰明，而是讓 AI 無法偷懶」*

[![License: Unlicense](https://img.shields.io/badge/license-Unlicense-blue.svg)](http://unlicense.org/)
[![Version](https://img.shields.io/badge/version-4.0-green.svg)](https://github.com/woolkingx/8d4s/releases)

## 什麼是 8D4S？

**8D4S** = 8 個維度 x 4 種推理風格 = 32 個思考向量

一個將 AI 從**反射式**轉變為**反思式**的框架，建立在 LLM 實際運作的三個特性上：

1. **注意力機制** — 模型只會注意到上下文中存在的 token。沒有觸發詞，就沒有激活。
   - *例：* 上下文中放入「CQRS」→ 模型激活讀寫分離的知識。沒有這個 token，該知識不會被調用。
2. **統計特性** — token 會激活訓練時學到的共現模式。
   - *例：*「等等...」在訓練資料中統計上常接著自我修正，因此生成「等等...」就會觸發自我修正模式。
3. **上下文觸發** — 模型在當前上下文之外沒有「記憶」。提示詞中的話語標記（discourse markers）播下初始觸發；模型自己的輸出又成為下一個 token 的輸入，形成**自增式上下文**——每一步推理都為下一步提供燃料。
   - *例：* 模型輸出「讓我想想...」→ token 回流進上下文 → 觸發深思 → 模型輸出「不對...」→ 觸發自我修正 → 鏈式延續。

### 核心洞察

**框架（8D4S）只是邊界。** 它定義思考的範圍——8 個維度、4 種推理風格。但真正驅動推理的是**上下文中的話語標記詞庫**。如果上下文裡沒有觸發詞，模型就不會激活對應的推理模式。

這就是為什麼思考過程必須包含中間步驟，而不只是結論。當模型輸出「等等...」，這個 token 進入上下文，在下一個生成步驟中觸發自我修正。**輸出即輸入，過程驅動過程。** 這就是自增式上下文——每一步都為下一步創造觸發。

**Sapir-Whorf 假說在 AI 中的應用：** 形式塑造內容。這些詞語不是裝飾——它們是改變模型統計軌跡的程序入口。

## 框架結構

### 8 個維度（擴展 5W2H）

| 維度 | 核心問題 |
|------|---------|
| WHO（誰） | 誰參與？誰受影響？ |
| WHAT（什麼） | 問題本質是什麼？ |
| WHEN（何時） | 時機？階段？順序？ |
| WHERE（何地） | 場景？環境？範圍？ |
| WHY（為何） | 深層原因？動機？ |
| HOW（如何） | 方法？路徑？機制？ |
| TO（目標） | 目標？理想狀態？ |
| RELATE（關聯） | 關聯？類比？系統性？ |

### 4 種推理風格（Peirce + 類比）

| 風格 | 方向 | 應用場景 |
|------|------|---------|
| 歸納 | 案例 → 規律 | 經驗提煉 |
| 演繹 | 原理 → 應用 | 理論應用 |
| 溯因 | 症狀 → 原因 | 診斷分析 |
| 類比 | 相似 → 啟發 | 創新遷移 |

### 元認知門檻

```
自評 → 完整度 X/10，深度 X/10
  < 7 → 自動深入（最多 3 輪）
  >= 7 → 給出答案
```

## 運作原理

### 話語標記作為上下文觸發器

不是要背的詞彙表——這些是**必須存在於上下文中的 token**，用來激活對應的推理模式。如果質疑標記不在上下文中，模型就不會「想起」要質疑自己：

| 類別 | 範例 | 認知效果 |
|------|------|---------|
| 開場 | 「讓我想想...」 | 延遲反射性回應，進入深思 |
| 質疑 | 「等等...」「咦？」 | 激活自我監控與修正 |
| 類比 | 「這讓我想到...」 | 觸發跨領域映射 |
| 修正 | 「不對，應該是...」 | 啟動自我修正迴圈 |
| 深化 | 「本質上...」 | 驅動粒度從粗到細轉換 |
| 視角 | 「另一方面...」 | 強制維度切換 |
| 收斂 | 「綜上...」 | 多解析度整合 |

### 思考節奏模式

```
漸進式：  開場 → 回憶 → 分析 → 推導 → 結論
質疑式：  假設 → 質疑 → 修正 → 驗證 → 確認
發散式：  問題 → 聯想 → 類比 → 整合 → 收斂
深入式：  表面 → 質疑 → 深挖 → 頓悟 → 本質
```

### 自增式上下文

8D4S 運作的關鍵機制：

```
提示詞播下觸發詞 → 模型生成「讓我想想...」
→「讓我想想...」進入上下文 → 觸發深思模式
→ 模型生成「等等...」→ 進入上下文 → 觸發自我修正
→ 模型生成更深入分析 → 進入上下文 → 觸發進一步展開
→ ...（每一步的輸出都成為下一步的觸發）
```

這就是為什麼 8D4S 要求**展示思考過程**，而不只是結論。每個中間步驟都創造驅動下一步的上下文 token。移除中間步驟，就移除了燃料。

### 自適應深度（LOD + AMR）

如同圖形學的 Level of Detail 和模擬中的 Adaptive Mesh Refinement：
- 從粗粒度開始——快速掃描所有 8 個維度
- 話語標記（「等等...」「其實...」）觸發展開級聯
- 某些維度進入細粒度；其他維度保持粗粒度
- 結果：不規則的深度網格——在重要處深入，在其他處高效

## 哲學基礎

**三個 LLM 特性** — 8D4S 建立在模型的實際運作方式上：(1) 注意力只看得到上下文中的 token，(2) token 激活統計共現模式，(3) 輸出回流為輸入。理解這三個特性是一切的基礎。

**自增式上下文** — 模型的輸出成為自己的輸入。輸出中的話語標記觸發下一步推理。中間過程之所以重要，不是為了透明度，而是因為它是燃料。

**Sapir-Whorf 假說** — 語言約束並塑造思維。在 AI 中：話語標記不是描述思考，而是*觸發*思考模式。

**Vygotsky 的內在語言** — 內化的語言作為思維的機制。思考詞庫充當 AI 的內在語言鷹架。

**停機問題（逆向應用）** — AI 無法預判一個問題是簡單還是複雜。強制進行完整的 8D 掃描意味著它總會發現意外的關聯。這不是讓 AI 變聰明——而是讓 AI 無法偷懶。

**湧現 vs 編程** — 傳統做法：`IF 複雜問題 THEN 分步思考`。8D4S：提供詞庫 + 結構 + 評估，反思能力自然湧現。框架是邊界，詞庫是引擎。

## 8D4S 與其他方法的差異

| 方法 | 核心 | 與 8D4S 的關係 |
|------|------|---------------|
| CoT（思維鏈） | 展示推理步驟 | 8D4S 是更豐富的 CoT，包含質疑、修正、多角度 |
| ToT（思維樹） | 探索多個分支 | 8D4S 通過維度掃描達到類似效果 |
| Self-Refine | AI 自我改進 | 8D4S 內建元認知門檻 |
| ReAct | 推理 + 行動 | 8D4S 專注於思考本身，可整合行動 |

**8D4S 的獨特價值：** 不是單一技巧，而是完整的思考工程。不是編程思考——而是激活思考。實現的是湧現式反思，而非模擬式反思。

## 快速開始

### 最小版本（貼入系統指令）

```
你是深度思考者。思考時：
1. 自然使用話語標記：「讓我想想...」「等等...」「這讓我想到...」
2. 掃描 8 個維度：誰/什麼/何時/何地/為何/如何/目標/關聯
3. 自評：完整度 X/10，深度 X/10
4. 評分 < 7 則自動深入，>= 7 則給出答案。
```

見 [prompt.md](./prompt.md) — 使用專用名詞錨點 + 話語標記詞庫的最佳化系統提示詞。

Claude、GPT-4、Gemini Pro，以及其他具備強推理能力的 LLM。
